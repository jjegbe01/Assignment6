AngularJS

The Go-To Javascript Library

Posted by Julian Egbert on February 11, 2015

So, what is AngularJS?

We'll start off with a formal definition. According to Angular's official documentation, it is defined as follows:

"AngularJS is a structural framework for dynamic web apps. It lets you use HTML as your template language and lets you extend HTML's syntax to express your application's components clearly and succinctly. Angular's data binding and dependency injection eliminate much of the code you would otherwise have to write."

In other words, its a widely used javascript library which allows for a lot of functionality, and saves you from having to write a lot of code.

Example

Lets look at an example, once again courtesy of Angular's official documentation.

In this example, we'll be building a very simple web based invoice form. It will contain:

  • A text field for quantity, which can be incremented.
  • A text field for costs, which can also be incremented.
  • A Total field, which will be calculated and updated in real time.

Normally, this project would take quite a bit of code to accomplish. However, through the use of AngularJS we can build it in just a few minutes by expanding our HTML's functionality.

  Edit in Plunker
index.html 
<div ng-app ng-init="qty=1;cost=2">
						  <b>Invoice:</b>
						  <div>
							Quantity: <input type="number" min="0" ng-model="qty">
						  </div>
						  <div>
							Costs: <input type="number" min="0" ng-model="cost">
						  </div>
						  <div>
							<b>Total:</b> {{qty * cost | currency}}
						  </div>
						</div>

Lets evaluate our example, one step at a time.

First, let us establish that we have two text fields: Quantity and and Cost. Each are created with an input tag, with a defined type of "number." Upon looking at our code, we can see that each of these fields is given the name "qty" and "cost" within something called "ng-model".

"ng-model" is our first example of what we call a Directive in Angular. Put simply, a directive is a marker attached to an HTML element that tells Angular how to treat it. In our case, the ng-model directive is used to bind an input, select, or textarea element to a property.

Lets move to the top of the code. At the top of the example, we see something called "ng-init." This is another directive, which allows us to evaluate the contents of a given property. Since it has access to our two text fields through the use of ng-model, we can use it to initiate those two text fields with default values. (qty = 1, cost = 2)

Finaly, we move to the end of the example. Towards the bottom, you'll see our Total field. Inside of a set of double curly braces "{{ }}", we see a mathematical statement: qty * cost. This is clearly the calculation needed to evaluate our previous two fields. However, by placing this inside of a set of double curly braces, we tell Angular that we want it to evaluate that expression for us. At the end of that statement, we see a pipe ( | ) followed by the word "currecny." By placing these within the braces, but after our equation, we're telling Angular to automatically display the results in a monetary format.
